树莓派安装 Hadoop-YARN 伪分布式环境

在树莓派上配置 Hadoop 环境和在 X86 机器上大致相同，只需多修改一个地方，下面将会介绍。

准备工作

  • 树莓派版本：1代
  • 操作系统版本：2015-05-05-raspbian-wheezy
  • 配置SSH免密码登陆了

下载二进制文件

本人选择2.7版本的Hadoop ，从北理的镜像下载

wget http://mirror.bit.edu.cn/apache/hadoop/common/hadoop-2.7.0/hadoop-2.7.0.tar.gz

解压二进制文件并放到树莓派的指定目录

解压

tar -xzvf hadoop-2.7.0.tar.gz

移动到指定目录

mv hadoop-2.7.0 ~/app/hadoop

修改 ～/app/hadoop/bin/hdfs

这也是在树莓派配置Hadoop与在x86机器配置唯一的不同之处，由于1代树莓派CPU为armv7 , 不支持JVM的server 模式，因此要修改～/app/hadoop/bin中一行脚本。

在1.*版本的Hadoop，要修改的脚本在 {HADOOP_HOME}/conf/hadoop 文件中。

vim ～/app/hadoop/bin/hdfs

找到

HADOOP_OPTS="$HADOOP_OPTS -server $HADOOP_DATANODE_OPTS"

将 -server 删除

HADOOP_OPTS="$HADOOP_OPTS $HADOOP_DATANODE_OPTS"

格式化NameNode

hdfs namenode -format

如果格式化成功，会提示

...
...
Storage directory /home/pi/hadoop/name has been successfully formatted.
...
Exiting with status 0
...

如果格式化失败 退出状态码为非0

启动HDFS

执行命令

start-dfs.sh

这个时候是无法启动的,会提示 JAVA_PATH 未找到,现在要在 ~/app/hadoop/hadoop_env.sh文件中找到

#export JAVA_HOME=....

将前面的注释符去掉,后面填写jdk路径

export JAVA_HOME=/usr/lib/jvm/java-8-openjdk-amd64

启动的时候JVM会提示，本地库有问题之类的提示，因为Hadoop官方的release的本地库是为x86cpu编译的，如果arm cpu的设备要运行本地库，需要我们自己用交叉编译工具重新编译本地库。暂且不用管它，只要在编程的时候不用对本地库有依赖的接口，目前我知道的是压缩文件会依赖本地库，其它大部分都不会依赖。

执行 jps 命令，查看Hadoop相关进程

至少会有 DataNode NameNode SecondaryNameNode 这三个进程在运行，如果少了任何一个都说明HDFS启动有错。
